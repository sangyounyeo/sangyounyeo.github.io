CSS Concepts - Position

What are the differences between relative, absolute, and fixed positioning?

Properties

Cascading Style Sheets or CSS as it is more commonly called, is a stylesheet language which gives instructions for the browser to show the HTML in a specific way.

In CSS, properties assign a specific behaviour or style to an HTML element such as colour, font-size, and position.

Position

The position property can be applied to an element to tell the browser which type of positioning method it should use to portray the element.

There are 5 main values the position property uses to control the position of elements on a web page:

  • static
  • relative
  • absolute
  • fixed
  • sticky

To assign a value to the position property in CSS, you place the value after "position:".

For example, if you wanted to apply an absolute value, you would type position: absolute;

Static

By default, the position will be static if it is not assigned another value.

Elements will not take in other positioning properties like "top", "right", "bottom", and "left" and will be positioned according to the normal flow of the document.

EXAMPLE

.example {
position: static;
}

Relative

Elements with the value relative are positioned according to the normal flow of the document.

However, different to the static position, can be moved "relative" to the normal position using "top", "right", "bottom", and "left" properties.

In the example below, the position of that element would be moved 10 pixels down as it adds 10 pixels to the top. You can think of it as it moving away the amount of units entered from the direction.

EXAMPLE

.example {
position: relative;
top: 10px;
}

Absolute

Elements of the positional value absolute do not follow the normal flow of the document and are positioned relative to the closest positioned ancestor if it has any or the block containing the element.

As it doesn't follow the flow of the document, it leaves spaces which are filled by other elements and can overlap elements.

The example below would show the position of the element to have moved 20px down from the top of the ancestor and right side of the element would be on the right most side of the ancestor as it moved 0 pixels from the right.

EXAMPLE

.relative-ancestor {
position: relative;
width: 200px;
height: 100px;
}

.example {
position: absolute;
top: 20px;
right: 0;
}

Fixed

Elements with the value fixed are positioned relative to the viewport or browser window. This means that it will remain "fixed" or in the same position even if the page is scrolled or the window is shrunk or enlarged.

Like all the other position values except for static, you can use the "top", "right", "bottom", and "left" properties to position the element

In the example below, the element would always be at the bottom right corner even if you scroll, shrink or enlarge the browser window. As the position of the element relative to the viewport, there is zero space between the element and the bottom and right side of viewport and the element will always be positioned at the bottom right of the viewport.

EXAMPLE

.example {
position: fixed;
bottom: 0;
right: 0;
}

Sticky

The position of an element that is sticky is dependent on the user's scroll position.

The behaviour of the element switches between relative and fixed. It behaves as "fixed" when a the offset threshold is met but until then it acts like "relative".

Assume that the element in the example begins at half way down the viewport from the top of the page. As the position property top is set to 0, as the user scrolls down the page the element moves up the viewport relative to the page, but when the element hits to the top of the viewport it begins to act as it is fixed and remains at the top of the viewport unless it the page is scrolled up to the point which it originally was positioned relative to the document.

EXAMPLE

.example {
position: sticky;
top: 0;
}
