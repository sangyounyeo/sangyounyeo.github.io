JavaScript Fundamentals

The Connection between HTML, CSS, and JavaScript

Imagine building a house. HTML forms the foundation, defining the structure and layout. It's like the blueprint of your web page. CSS acts as the interior designer, adding colours, styles, and visual enhancements. JavaScript, on the other hand, is the handyman that brings the house to life by handling dynamic aspects and interactivity. It's like adding the plumbing, electricity, and other functional elements that make the house engaging and functional.

Control Flow and Loops

The relationship between control flow and loops can be seen as the baking process. The process involves following a recipe step by step and performing repetitive actions. Control flow in programming is akin to the recipe instructions, ensuring tasks are executed in a specific order. Loops, on the other hand, are like whisking ingredients in a bowl. You continue whisking until the mixture reaches the desired consistency. Similarly, in programming, loops allow you to repeat a set of instructions until a specific condition is met, just like whisking until the desired consistency is achieved.

EXAMPLE

let isRecipeReady = true;
let whiskedConsistency = false;

if (isRecipeReady) {
console.log("Start baking process!");

for (let i = 1; i <= 3; i++) {
console.log("Whisking the mixture...");

if (i === 3) {
whiskedConsistency = true;
}
}
}

Understanding the DOM

The Document Object Model (DOM) is a virtual representation of a web page. Let's imagine you're a chef in a busy restaurant, and the menu is your DOM. You interact with it to access information, make changes, and respond to customer requests. In web development, JavaScript enables you to manipulate the DOM, dynamically updating content, modifying styles, and adding interactive elements. It's like updating the menu to showcase daily specials or taking orders from customers.

EXAMPLE

let heading = document.getElementbyID("todaysSpecial");
heading.textContent = "Spaghetti";
heading.style.color = "red";

Accessing Data from Arrays and Objects

Arrays and objects can be described to be like a recipe book and a pantry, respectively. An array resembles a recipe book, containing multiple recipes (elements). You can access each recipe by referring to its page number (index) or search for specific recipes based on criteria. On the other hand, an object is like a well-organized pantry, with various ingredients (properties). You access specific ingredients by their names (keys) and retrieve their corresponding values. Arrays and objects offer distinct ways to store and retrieve data, depending on your requirements.

EXAMPLE

// Array
let recipe = ["Kimchi Fried Rice", "Lasagne", "Sushi"];
console.log(recipe[0]); // Output is Kimchi Fried Rice

// Object
let ingredient = {
name: "Carrot",
type: "Vegetable",
expiresIn: 10
};
console.log(ingredient.name); // Output is Carrot

The Power of Functions

Functions in programming can be compared to a set of instructions or a routine. They are blocks of code that perform a specific task when called. They help break down complex tasks into smaller, manageable parts, promoting reusability and modularity. Functions can receive inputs (arguments), execute operations, and return outputs, making them invaluable tools for problem-solving and organizing code.

EXAMPLE

function thanks(name) {
console.log("Thank you, " + name + "!");
}

thanks("John"); // Output is Thank you, John!
